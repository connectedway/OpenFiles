Open Files 5.0.1
Multi-Platform Event-Driven Application Framework
Open Files Software Framework

Open Files is a multi-platform, real-time, event-driven, high performance, and small footprint subsystem for application development.

This document details the structure and APIs available to platform integrators for initializing, configuring, starting up, and monitoring the Open Files stack. This document also details the various APIs utilized by the Open Files stack to interface with the platform specific routines (i.e. the porting layer).

The Open Files Documentation leverages the doxygen documentation generation tool. Doxygen arranges the product documentation in seven or more sections. To make navigation more intuitive, Open Files has added this main page and a collection of subpages that attempt to guide a reader through the APIs and relates the APIs to architectural constructs and workflow interests.

This document has three target audiances: those wishing to port Open Files to a target platform, those wishing to integrate an Open Files package to a target system, and those wishing to develop applications that use the functionality provided by Open Files.

  • Introducing Open Files Who is Open Files for? What kind of things can I do with it? How do I work with it? What else can it do? These questions and more will be answered in the introduction.
  • Accessing the Source Code. Open Files is not destributed as a binary for target platforms although it could be. As such, there isn't an instllation step. That means the first step in using Open Files is to download or cloan the Open Files repository.
  • Configuring a Platform Build Open Files comes standard with support for Windows, Linux, MacOS, Android iOS, ThreadX, and NucleusPLUS. Default configurations for these platforms are provided. It is also possible to create configurations for new platforms or to customize existing configurations.
  • Building the Open Files Stack Open Files uses the cmake build system. We recommend CLion (https://www.jetbrains.com/clion/) as a multi-target cmake aware IDE but using any properly configured cmake tool for your host system should work.
  • Running Generated Test Applications Open Files is essentially a framework or library that you can link with an application to generate an executable. Open Files provides a collection of test applications that are products of the build.
  • Developing with Open Files Open Files provides a full set of APIs that allow developers to implement robust multi-platform, asynchronous or threaded applications. The facilities provided are discussed in this section.
  • Open Files Asynchronous Programming Model A major feature of open files is it's support for asynchronous concurrency model. To learn more about the async programming model, read here.
