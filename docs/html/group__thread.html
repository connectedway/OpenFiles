Detailed Description

The thread facility allows Open Files to manage platform threads in a generic fashion.

For examples on using the threads see test_thread.c

Create a Thread on the target platform

The routine will create a thread on the target platform. Typically, there are many parameters that can be specified when creating target platform threads but, for the most part, Open Files cares only about a few. For simplicity, we have limited our support only to those parameters required.

Parameters
schedulerThe Entry point to the thread. The entry point will be passed the thread handle and a pointer to the context.
contextThe context to pass to the thread
detachstateWhether the thread should clean up on it's own or whether someone will join with it
Returns
The thread handle
Examples
test_thread.c.

◆ ofc_thread_create_local_storage()

OFC_CORE_LIB OFC_VOID ofc_thread_create_local_storage ( OFC_VOID  )

Create Local Storage for Local Variables

This routine initializes local storage for a thread. This routine is only required on platforms that do not support the notion of Thread Local Storage. It is a noop on platforms that do support TLS. The routine should be called before any call using the Open Files API (any call that manipulates last error).

Examples
test_file.c.

◆ ofc_thread_create_variable()

OFC_CORE_LIB OFC_DWORD ofc_thread_create_variable ( OFC_VOID  )

Create a Thread Specific Variable

It is often useful to create multiple threads all running the same program. In this case, it is also useful to create variables that are specific to the thread. Therefore, two threads will have variables that are specific only to that thread. This is typically used for C runtime functions involving errno or GetLastError on Win32.

Returns
Variable ID

◆ ofc_thread_delete()

OFC_CORE_LIB OFC_VOID ofc_thread_delete ( OFC_HANDLE  hThread)

Delete a thread

This routine will cause the target thread to begin deletion. It doesn't actually forcibly kill the thread, but rather sets a flag that the thread can check by calling ofc_thread_is_deleting. If this flag is set, the thread should clean up and exit.

Parameters
hThreadHandle to the thread to delete
Examples
test_thread.c.
