Detailed Description

Provides network socket level abstraction and helper routines

There are two levels to the network abstraction. A 'socket' layer and an 'implementation' layer. The socket layer is platform independent and is used to perform common management of network sockets. The implementation layer is platform dependent and interfaces with the platforms socket calls.

If a platform has a socket model, then it is recommended that an implementation layer be created to interface with that socket layer. If a platform does not use a socket model for communication, it is recommended that the Open Files 'socket' layer be replaced with a network layer that interfaces directly to the platform network layer.

enum OFC_SOCKET_EVENT_TYPE

Socket Event Types

All open sockets are through an associated event. When an event fires, it can be for any number of reasons. Applications may poll the socket's event to determine the specific event type.

Not all these event types will be genererated in the current implementation

Enumerator
OFC_SOCKET_EVENT_CLOSE 

The socket has been closed

OFC_SOCKET_EVENT_ACCEPT 

An incoming connection request has been received on a socket that has been listening for connections

OFC_SOCKET_EVENT_ADDRESSCHANGE 

The underlying IP address has changed for the socket

OFC_SOCKET_EVENT_QOS 

A Quality of Service Event has occurred. This may be reported by the lower level platform specific network code.

OFC_SOCKET_EVENT_QOB 

Out of Band data has been received.

OFC_SOCKET_EVENT_READ 

Data has been received and is available

OFC_SOCKET_EVENT_WRITE 

Buffer space is available to write data for transmission

Socket Types Supported by Open Files.

Enumerator

Unspecified Socket Type (Do not use)

SOCKET_TYPE_STREAM 

Socket is a Stream Type. Usually indicates TCP. Bytes are transferred as a stream of bytes. Any packet delimination is the responsibility of higher layer protocols and typicaly is based on the byte position of the stream.

SOCKET_TYPE_DGRAM 

Socket is a Datgram Type. Messages are grouped into packets without any guarantee of delivery.

Socket is a NetBIOS Session Type

SOCKET_TYPE_ICMP

Create a Stream socket and connect to a remote target

This call will create a TCP connection to the specified port on the remote IP. All sockets are set as non-blocking. All parameters should be specified using the platform endianess.

Parameters
ipPointer to the remote IP to connect to.
portThe port on the remote to connect to.
Returns
Handle to the created socket or OFC_HANDLE_NULL if creation failed. This call is non-blocking so on return, the connection may not be complete. When the socket is connected, an event will be set for the handle and a call to ofc_socket_connected will return success.

Create a datagram socket

This call will create a UDP datagram socket and assign it to the interface of the specified ip address with the specified port.

Parameters
ipIP of the interface that the datagram socket should be assigned to. The datagram socket will receive incoming datagrams from this interface. If the ip is OFC_INADDR_ANY, then this socket will accept datagrams from any interface.
portPort to accept incoming datagrams from
Returns
A handle to the created datagram socket. If OFC_HANDLE_NULL is returned, then the datagram creation failed. The socket is non-blocking so this will return immediately. Events will be generated when traffic arrives or buffer space is available.

OFC_CORE_LIB OFC_HANDLE ofc_socket_listen ( const OFC_IPADDR *  ip,
OFC_UINT16  port 
)

Create a TCP socket and listen for incoming connections

This call will listen for incoming connections on the interface specified by the ip address and on the port specified by the port. All arguments are specified in the platform endianess.

Parameters
ipThe ip for the interface to listen for connections on. Can be OFC_INADDR_ANY to listen for connections on any interface.
portThe port to listen for incoming connections on.
Returns
A handle to the created socket. If OFC_HANDLE_NULL, socket creation failed. The call is non-blocking so it will return immediately. When an incoming connection arrives, a socket event will be generated. A subsequent call to ofc_socket_accept can then be issued.
Examples
test_stream.c.
